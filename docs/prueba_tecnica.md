[FULLSTACK]Desarrollo del MVP "NanoBank Ledger" para gestión financiera en examen técnico Full Stack Senior. ID de Evaluación: FS-SR-2026-002 Tiempo Límite: 3 Horas (Estricto) Horario de Entrega: Hoy antes de las 3:00 PM 📝 1. DESCRIPCIÓN DEL RETO Desarrollar el MVP de "NanoBank Ledger", un sistema de gestión de movimientos financieros y presupuestos. El objetivo es evaluar la capacidad del ingeniero para manejar integridad de datos, arquitecturas desacopladas y la eficiencia en el uso de IA para garantizar un producto testeado y profesional en tiempo récord. 🛠 2. STACK TECNOLÓGICO Backend: Java 21+ con Spring Boot 3.x. (PLUS: Uso de Kotlin). Frontend: Angular 17+ (Uso de Standalone Components y Signals obligatorio). Base de Datos: LIBRE ELECCIÓN (SQL o NoSQL). El candidato debe justificar su elección en el README (ej. PostgreSQL, DynamoDB, MongoDB, H2, etc.). IA: Uso estratégico de herramientas generativas (Copilot, ChatGPT, etc.). 📋 3. REQUISITOS FUNCIONALES (OBLIGATORIOS) 1. Dashboard de Billeteras: Crear y listar diferentes "Billeteras" (ej. Ahorros, Gastos, Inversiones). 2. Gestión de Transacciones: Registrar ingresos y gastos vinculados a una billetera específica. 3. Interfaz Dinámica: Implementar un filtro en tiempo real por categoría o fecha. Funcionalidad de Drag & Drop: Mover una transacción de una billetera a otra (actualizando saldos en el backend). 4. Seguridad: Implementar un esquema de Auth JWT para proteger los recursos. 🏗 4. REQUISITOS TÉCNICOS DE INGENIERÍA (OBLIGATORIOS) A. Arquitectura y Escalabilidad Principios SOLID: Código orientado a objetos con alta cohesión y bajo acoplamiento. Patrones de Diseño: Uso obligatorio de DTOs y un Global Exception Handler en el Backend. Justificación de Datos: Explicar brevemente por qué eligió esa base de datos y cómo estructuró el modelo para soportar escalabilidad.B. Calidad y Pruebas (Unit Testing) Cobertura (Coverage): Es obligatorio alcanzar el 80% en ambas capas. Validación: Los tests deben cubrir la lógica de cálculo de saldos y la transferencia entre billeteras. Herramientas: JUnit 5 / Mockito y Jasmine/Jest para Angular. C. Skill de IA (Prompt Engineering) En el archivo README.md es obligatorio incluir: Bitácora de Prompts: Registro de los prompts clave utilizados. Criterio Senior: Justificar qué sugerencias de la IA fueron modificadas por no cumplir con SOLID o por ser ineficientes. 🚦 5. CRITERIOS DE EVALUACIÓN Escalabilidad y Diseño (30%): Calidad de la arquitectura y aplicación de SOLID. Calidad de Pruebas (30%): Robustez de los tests y cumplimiento del 80% de coverage. Dominio Técnico (20%): Tipado estricto, manejo de Signals y elección de base de datos. IA Mastery (20%): Capacidad de supervisión técnica y rapidez en la entrega asistida.
